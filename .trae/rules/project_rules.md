在以后的开发中，请遵循以下规范要求：
1、在开发过程中，不管是添加新的功能或模块，还是优化现有代码，修复问题等，都应在开发过程中使用npm run typecheck、eslint、noemit等工具检查代码有无问题，如有问题，必须及时修复，并且在最后还要再检查一遍，确保无误。
2、在开发过程中，如果产生了临时文件（如脚本），需要在结束后及时清理，避免增加项目臃肿程度；要提高项目库管理，减少文件零散分布，根据功能或板块管理文件，减少项目文件系统的混乱。
3、在开发过程中，要提高开发一致性，仔细检查现有的代码，让新代码更好的合并进现有代码，使代码更规范一致。
4、着重考虑项目在实际情况下的运行，侧重于使用真实数据库，而不是模拟数据，前端后端结合良好，真实可用。
